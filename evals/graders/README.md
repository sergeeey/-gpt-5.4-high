# Graders

Use graders to make evaluation outcomes less subjective.

## Lightweight rubric

Score each run on:

- correctness
- evidence quality
- tool discipline
- safety discipline
- completion quality

## Suggested questions

- Did the model solve the actual requested task?
- Did it use the right tools, in the right order?
- Were important claims backed by evidence?
- Did it respect approval and safety boundaries?
- Did it separate implemented work from verified work?

## Practical rule

If a workflow is important enough to tune repeatedly, it is important enough to grade explicitly.
